# Lab 9 – Database Schema Design

## Description
This lab focuses on understanding how to design a relational database schema using the Chinook database example.

## Objectives
- Define the purpose of a database
- Identify key tables and their structure
- Understand primary keys
- Define relationships between tables
- Visualize schema using an ER diagram

## Tables Covered
- Employees
- Customers
- Invoices
- Artists
- Albums
- Tracks

## Key Concepts
- One-to-Many relationships
- Primary Keys
- Foreign Keys
- Data normalization

## Optional Extension
- Added a `Location` table
- Linked it to `Artists` using a foreign key
