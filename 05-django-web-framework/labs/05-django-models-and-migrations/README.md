# Lab 05: Models and Migrations in Django

## Introduction
In this lab, a Django model was created and database migrations were applied to reflect changes in the database structure. This helped demonstrate how Django manages data models using its ORM system.

## Goal
To create a model, apply migrations, and track changes in the database.

## Objectives

- Create a Django model called `Drinks`.
- Define model fields for menu items.
- Register the model in the Django admin site.
- Perform migrations to create and update database tables.
- Understand how Django tracks schema changes.

## Project Structure

- `myapp/models.py` → Defines the `Drinks` model
- `myapp/admin.py` → Registers the model in Django admin
- `myapp/migrations/` → Contains migration files
- `db.sqlite3` → SQLite database generated by Django

## Key Concepts

- Django models
- ORM (Object-Relational Mapping)
- Migrations (`makemigrations`, `migrate`)
- Database schema evolution
- Django admin registration

## Model Created

Example model added to `myapp/models.py`:

```python
from django.db import models

class Drinks(models.Model):
    drink_name = models.CharField(max_length=200)
    price = models.IntegerField()

    def __str__(self):
        return f"{self.drink_name} - {self.price}"
```

## Admin Registration

Register the model in `myapp/admin.py`:

```python
from django.contrib import admin
from .models import Drinks

admin.site.register(Drinks)
```

## Commands Used

```powershell
python manage.py makemigrations
python manage.py migrate
python manage.py showmigrations
```

## Final Result

- A `Drinks` table was successfully created in the database.
- Migrations were applied and tracked by Django.
- The model is now registered in the admin site.
